1.1 HTML FOUNDATION

The foundation of just about every page written in html boils down to four simple tags. When used together, they define where your code begins and ends, what should be displayed, and where. Those tags are:


<html> <head> <title> and <body>.



In the box below are these four tags in context, and then a break down in the simplest terms of what each one does.

<HTML>

The <html> tag functions as a declaration to the browser, saying: "Everything between the <html> and </html> tags is coded in html, and you should interpret it that way." Simple!

<HEAD>

The <head> tag has always been of the more headscratch-y ones for me.

At a glance, it doesn't seem to do much of anything. What you put in this section isn't generally displayed on the page, so what the shit is it for? Without going into too much detail, it's a spot in your code where only certain tags belong.

Think of it as a little section for putting statements that only the browser needs to see. The next tag is an example of one of the most important tags in that category.

<TITLE>

The <title> tag is another fairly straightforward one. It declares the title of the page to the browser. Whatever you put in this section is what shows up in the browser tab and the taskbar when the browser window is minimized into it.

<BODY>

And lastly we come to the <body> tag. This is another mercifully straightforward one! Short of it is that this is the tag where everything that appears on the page belongs. Text, images, tables, lists, and the rest of it. If you want the user to see it when they open up your webpage, this is where it goes.

WRAPPING THINGS UP

There you have it!

Even though these are all pretty simple concepts, I know it can still seem like a lot to sit down in front of and absorb all at once. If you're feeling overwhelmed or overburdened by information, don't worry about it. All of those explanations don't really need to click just yet.

All you need to understand is that you will use these four tags in the code of every html webpage you make and, when boiled down to it's simplest form, every html page looks more or less exactly like this:
<html>

<head>
<title>PAGE TITLE</title>
</head>

<body>
PAGE CONTENT
</body>

</html>
