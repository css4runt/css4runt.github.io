1.1 HTML FOUNDATION

The foundation of just about every page written in html boils down to four simple tags. Those tags are:


<html> <head> <title> and <body>.



When used together, these four tags define where your code begins and ends, what should be displayed, and where. In the box below are these four tags in context, and then a break down in the simplest terms of what each one does.
<html>

<head>
<title> PAGE TITLE </title>
</head>

<body>
PAGE CONTENT
</body>

</html>

<HTML>

The <html> tag functions as a declaration to the browser, saying: "Everything between the <html> and </html> tags is coded in html, and you should interpret it that way." Simple!





<HEAD>

The <head> tag has always been of the more headscratch-y ones for me.

At a glance, it doesn't seem to do much of anything. What you put in this section isn't generally displayed on the page, so what the shit is it for? Without going into too much detail, it's a spot in your code where only certain tags that communicate certain information about the page belong.

Think of it as a little section for putting certain declarations that only the browser needs to see. The next tag is an example of one of the most important ones in that category.





<TITLE>

The <title> tag is another fairly straightforward one. It declares the title of the page to the browser. Whatever you put in this section is what shows up in the browser tab and the taskbar when the browser window is minimzed into it.
